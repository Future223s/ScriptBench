# Workflow Builder

![[Pasted image 20260711115801.png]]

UX and information architecture issues:
1. Weak visual hierarchy
	- Everything has roughly the same visual weight
	- No step-by-step flow is immediately apparent
2. Large amounts of space are devoted to documentation
3. Too much vertical space is consumed before work begins
4. The left form feels like it belongs to a separate application
5. Canvas should be the hero / primary workspace
	- Large white margins and thick border make it look like another card rather than the primary workspace
6. Navigation cards create ambiguity 
	- They resemble feature cards
	- Traditional tabs communicate navigation much more clearly
7. Canvas controls lack affordance
	- If pills are interactive they should look interactive
	- If they're legends they should look passive.
	- Controls don't strongly communicate what the user can do next
8. Repetition
	- Section names all begin with "Workflow"
	- Important differentiators get buried
	- Instead: Builder, Details, Canvas,

# Workflow Step Assignment Popup

![[Pasted image 20260711115710.png]]

1. Main workflow is ambiguous
	- No indication of which path is active
	- Solution: Two choices should be mutually exclusive
	- Top of the modal should feature tabs so that only controls associated with the selected option are shown
2.  Primary action semantics
	- "Confirm Step Selection" sounds long and mechanical
	- Solution: Use a more outcome-oriented label like "Add step"
3. Unused whitespace
	- Makes the modal feel taller and heavier than necessary.
